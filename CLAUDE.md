## Project Overview

**Torre Central — Autolab** is a single-file SLA monitoring dashboard for automotive repair shop management. The entire application lives in one HTML file (`Torre Central — Autolab.html`) with embedded HTML, CSS, and JavaScript — no build system, no framework, no backend.

**Runtime dependencies (CDN-only):**
- `xlsx.full.min.js` (v0.18.5) — Excel export
- Google Fonts (JetBrains Mono, Syne)

## Running the Application

Open `Torre Central — Autolab.html` directly in a browser. No build step, no server needed.

**First-run setup required:**
1. Enter your name
2. Paste a Google Apps Script deployment URL (from Google Sheet → Extensions → Apps Script → Deploy → Manage Deployments)

Configuration is persisted in `localStorage` under key `torre_cfg`.

## Architecture

### Data Flow

```
Google Sheet + Apps Script
        │ JSONP (_dataCb callback)
        ▼
allVehicles[]          ← raw records
        │ processData()
        ▼
filteredVehicles[]     ← after multi-select filters applied
        │
        └── renders into 7 views
```

### State (top of `<script>` block, ~line 663)

- `CFG` — user config (name, API key/URL)
- `allVehicles[]` — raw vehicle records from Apps Script
- `filteredVehicles[]` — filtered subset
- `recoItems[]` — recommendation items
- `msState` — multi-select filter state (`flota`, `taller`, `servicio`, `fase`)

### Core Functions

| Function | Purpose |
|---|---|
| `loadData()` | JSONP fetch to Apps Script; triggers every 300s |
| `_dataCb(data)` | JSONP callback; receives raw rows |
| `processData()` | Calls `detectPhase()` + `detectAlert()` on each vehicle, applies filters |
| `detectPhase(v)` | Assigns `v._phase`: `diagnostico`, `cotizacion`, `ejecucion`, `entrega` |
| `detectAlert(v)` | Assigns `v._alert`: `lagging`, `leading`, or `ok`; sets `v._alertMsg` |
| `renderView()` | Routes to the active view renderer |

### Views (7 tabs)

- **Torre** — real-time SLA radar with color-coded vehicle cards (red=lagging, blue=leading, green=ok)
- **Cotizaciones** — quote pipeline and quote metrics
- **Partes** — parts inventory and waiting status
- **Asesores** — advisor performance rankings and workload
- **Tareas** — task management (create, assign, track)
- **Métricas** — KPI dashboards
- **Recomendaciones** — auto-generated action items from SLA violations

### Filtering

Multi-select toggle buttons in the top bar filter by `flota`, `taller`, `servicio`, and `fase`. Changes re-run `processData()` and re-render the active view.

### Task & Recommendation Engine

Recommendations are generated by analyzing each vehicle's `_phase`, `_alert`, and time-in-phase. They are stored in `recoItems[]` and can be converted to tasks (stored in localStorage). Tasks are assigned by advisor name.

### Persistence

All client-side state (config, tasks, notes) is stored in `localStorage`. Nothing is written back to Google Sheets.

## Vehicle Record Shape

Key computed fields added by `processData()`:

```javascript
{
  _phase: 'diagnostico' | 'cotizacion' | 'ejecucion' | 'entrega',
  _alert: 'lagging' | 'leading' | 'ok',
  _alertMsg: string,  // human-readable SLA status
}
```

Raw fields from the Sheet include: `placa`, `asesor`, `taller`, `SERVICIO`, `Estado SLA`, `SLA 3`, `dias habiles desde recepcion`, `pct_sla_quote_teoric_expiration`, `pct_sla_entrega`, etc.
